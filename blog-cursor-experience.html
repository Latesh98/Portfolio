Building My Portfolio with Cursor: A Developer's Experience

June 2025 • 6 min read
Summary: This post shares my journey of creating a personal portfolio website using Cursor, an AI-powered code editor. I'll explain what Cursor is, how it streamlined my workflow, and why it's a great tool for web development and learning.

What is Cursor?

Cursor is a modern code editor that integrates AI directly into your development workflow. It's built on top of VS Code, but with powerful features like:

  • AI pair programming: Get real-time code suggestions, explanations, and bug fixes.
  • Natural language commands: Ask for code changes, refactoring, or explanations in plain English.
  • Context awareness: Cursor understands your project structure, dependencies, and even your coding style.
  • Seamless integration: Use it for web, Python, Node.js, and more—no extra setup required.

Why I Chose Cursor for My Portfolio

  • Speed: I could describe what I wanted ("add a blog section," "make links animated on hover," "remove dark mode toggle") and Cursor would instantly update my HTML/CSS/JS.
  • Learning: Whenever I was unsure about a concept (like semantic HTML, responsive design, or GitHub Pages deployment), I could ask Cursor for an explanation or code sample.
  • Error reduction: Cursor caught typos, missing tags, and even suggested accessibility improvements.
  • Experimentation: I could try new features (like animated backgrounds or tooltips) with confidence, knowing I could always undo or refine changes.

How I Built My Portfolio with Cursor

  1. Project Setup: I started with a basic HTML/CSS template. Cursor helped me organize my files and set up a clean structure.
  2. Navigation & Pages: I described the sections I wanted (Home, About, Projects, Blog), and Cursor generated the navigation bar and individual pages.
  3. Styling & Interactivity: I asked Cursor to add hover effects, tooltips, and even a live animated background—all with smooth transitions and modern design.
  4. Blog System: Adding new blog posts was as simple as describing the content and images. Cursor handled the formatting, linking, and even semantic HTML.
  5. Deployment: Cursor guided me through deploying to GitHub Pages, including creating a workflow file and writing a README with instructions.
  6. Iterative Improvements: Whenever I wanted to tweak something (like removing the skills section or updating contact info), I just told Cursor what to do.
Tips for Using Cursor Effectively:
  • Be specific: The more clearly you describe your goal, the better Cursor can help.
  • Iterate: Don't be afraid to ask for changes, reversions, or new features.
  • Learn as you go: Use Cursor's explanations to deepen your understanding of web development.
  • Leverage AI for repetitive tasks: Let Cursor handle boilerplate, formatting, and refactoring.

Conclusion

Building my portfolio with Cursor was a game-changer. It felt like having an expert developer by my side, ready to code, explain, and improve my project at any time. If you're looking to build your own site, learn new skills, or just code faster, I highly recommend giving Cursor a try!
